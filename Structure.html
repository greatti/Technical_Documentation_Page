What is Github?

In a few words, github is a platform for the development of group projects. Hub is a word used to indicate a platform that connects computers to a network to share information

Github is also available for companies, that one is called GitHub Enterprise, where we find a intelligent wat to manage group work.

The best thing in Github is that a lot of people of the group can work the same project at the same time in their own computer, all around the world, without necessarily compromising the original work, that is, modifying it as a "suggestion"

To create your account, enter GitHub and fill in the requested data - name, email and password. In a few minutes, you will receive a confirmation email to start using the platform.

What you should know about Github

Github have some functions to facilitate your work, some of them you can read below:

  1. README
    1. Is the first impression one would have about a project or profile. Its a set of information necessary for readers to understand, it works like a manual writen in Markdown that we'll see later.
  2. Repository
    1. A repository contains all of your project's files and each file's revision history. You can discuss and manage your project's work within the repository. You can own repositories individually, or you can share ownership of repositories with other people in an organization.
  3. Projects
    1. It is also one of the Resource in Git Repository and main use of it is to manage the projects with a visual board. If you create a project in Git Repository it create a visual board like a Kanban board to manage the project.
  4. Branch
    1. A branch is a copy of a repository. You can use it to develop your project individually, without changing the original directory. Thus, only after it is finalized, will it be combined again with the original repository.
    2. Essa função é útil, por exemplo, se você tem mais de uma opção para a finalização de um projeto. Você pode testar as opções separadamente e incluir o código definitivo só quando você decidir qual alternativa é a melhor.
  5. Pull Request
    1. A Pull Request is when you inform other members of your team that you are going to incorporate your branch into the original repository. Team collaborators may or may not accept the Pull Request.
What you should know about Git

Git is a software to generate projects where a lot of devs can work together at the same time, and everytime someone changes some code, is generated other version for the project, so there is no risk to lose all the progress

  1. CONFIG
    1. One of the most used git commands is git config which can be used to set user-specific configuration values ​​such as email, preferred diff algorithm, username and file format etc.
  2. INIT
    1. This command is used to create a new GIT repository.
  3. ADD
    1. can be used to add files to the index.
  4. PUSH
    1. envia as alterações feitas para o ramo mestre do repositório remoto associado ao diretório de trabalho.
  5. COMMIT
    1. is used to confirm changes in the head. Note that any changes made will not go to the remote repository.
  6. STATUS
    1. displays the list of changed files along with files that have not yet been added or confirmed.
  7. PULL
    1. To merge all changes present in the remote repository to the local working directory, the pull command is used.
  8. MERGE
    1. is used to merge a branch into the active branch.
